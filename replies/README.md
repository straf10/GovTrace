# `replies/` — δικαίωμα απάντησης (P2-13)

Το [DISCLAIMER](../docs/DISCLAIMER.md) υπόσχεται: αν φορέας καταθέσει
απάντηση/εξήγηση για δείκτη που τον αφορά, δημοσιεύεται **αυτούσια, δίπλα
στον δείκτη**, με ημερομηνία και ιδιότητα αποστολέα, χωρίς λογοκρισία πέραν
των αναγκαίων επισημάνσεων πλαισίου.

Αυτός ο φάκελος είναι η **μοναδική πηγή αλήθειας** για τις απαντήσεις αυτές.
Είναι top-level (ΟΧΙ μέσα στο `data/`) επίτηδες: το `data/` είναι εξ ολοκλήρου
εκτός git (pipeline artifacts) και το git δεν επιτρέπει re-include αρχείου
μέσα σε ήδη αποκλεισμένο parent directory. Αυτός ο φάκελος **είναι tracked**
γιατί είναι χειροκίνητα επιμελημένο, νομικά ευαίσθητο περιεχόμενο, όχι
παραγόμενα δεδομένα.

## Διαδικασία

1. Ο φορέας στέλνει απάντηση (email/επιστολή) για συγκεκριμένο δείκτη/έτος.
2. Χειροκίνητος έλεγχος: όχι παράνομο περιεχόμενο (βλ. DISCLAIMER §4). ΔΕΝ
   γίνεται λογοκρισία/σχολιασμός πέραν αυτού.
3. Δημιουργείται/ενημερώνεται `replies/<ΑΦΜ>.json` (βλ. σχήμα
   `_TEMPLATE.json`).
4. Commit + επόμενο `python scripts/build_foreas_data.py` run ενσωματώνει
   την απάντηση στο `foreas_pages.json` -> εμφανίζεται στο `/foreas/<ΑΦΜ>/`.

## Σχήμα

Ένα αρχείο ανά ΑΦΜ (`<ΑΦΜ>.json`), όνομα αρχείου = κανονικοποιημένο ΑΦΜ.
`replies` είναι πίνακας ώστε να χωράνε πολλαπλές απαντήσεις στον χρόνο.
`text` δημοσιεύεται ως απλό κείμενο (χωρίς HTML) — αυτούσιο, μη επεξεργάσιμο.

Αρχεία που ξεκινούν με `_` (π.χ. `_TEMPLATE.json`) αγνοούνται από τον loader.
